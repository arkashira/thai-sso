## 1. `docker-compose.yml`

```yaml
# docker-compose.yml
# ------------------------------------------------------------
# Orchestrates the Thai‑SSO stack (FastAPI app, Postgres, Redis, MailHog)
# ------------------------------------------------------------
version: "3.9"

services:
  web:
    build:
      context: .
      dockerfile: docker/Dockerfile
    container_name: thai_sso_web
    restart: unless-stopped
    env_file:
      - .env
    environment:
      - PYTHONUNBUFFERED=1
    depends_on:
      - db
      - redis
      - mailhog
    ports:
      - "8000:8000"
    command: uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload

  db:
    image: postgres:15-alpine
    container_name: thai_sso_db
    restart: unless-stopped
    environment:
      POSTGRES_USER: ${POSTGRES_USER:-sso_user}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD:-sso_pass}
      POSTGRES_DB: ${POSTGRES_DB:-sso_db}
    volumes:
      - db_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"

  redis:
    image: redis:7-alpine
    container_name: thai_sso_redis
    restart: unless-stopped
    ports:
      - "6379:6379"

  mailhog:
    image: mailhog/mailhog:latest
    container_name: thai_sso_mailhog
    restart: unless-stopped
    ports:
      - "1025:1025"   # SMTP
      - "8025:8025"   # Web UI

volumes:
  db_data:
```

> **Why this version?**  
> * Keeps the original FastAPI‑based stack (candidate 1) while adding the optional `mailhog` service from candidate 2 for local email testing.  
> * Uses `${VAR:-default}` syntax so that the stack can run out‑of‑the‑box with sane defaults, yet still respects values from a real `.env` file.  
> * `env_file: - .env` pulls the variables defined in `.env.example` (once copied to `.env`).
